🚀 QUICK START:

Option 1 (Automated - 1 minute):
  $ sudo bash SETUP.sh

Option 2 (Manual - 2 minutes):
  $ sudo apt-get install nodejs npm
  $ cd /root/tribal-chief-panel
  $ npm install
  $ sudo node server.js

Option 3 (Docker - 1 minute):
  $ docker run -d -p 8080:8080 \
      -e DB_PATH=/opt/tribal/database/chief.db \
      -v /opt/tribal:/opt/tribal \
      tribal-chief-panel

Then open: http://localhost:8080

❓ TROUBLESHOOTING:

Panel won't start?
  → Check: sudo journalctl -u tribal-chief-panel -n 50
  → Verify: node --version
  → Test: sudo node /root/tribal-chief-panel/server.js

Can't access panel?
  → Test locally: curl http://127.0.0.1:8080/api/health
  → Check firewall: sudo ufw status
  → Verify service: sudo systemctl status tribal-chief-panel

Database errors?
  → Check file: ls -la /opt/tribal/database/chief.db
  → Test access: sqlite3 /opt/tribal/database/chief.db ".tables"
  → Fix permissions: sudo chmod 666 /opt/tribal/database/chief.db

Users not appearing?
  → Check DB has data: sqlite3 /opt/tribal/database/chief.db "SELECT COUNT(*) FROM users;"
  → Restart service: sudo systemctl restart tribal-chief-panel
  → Check logs: journalctl -u tribal-chief-panel -f

📈 SCALING INFORMATION:

Current Setup (SQLite):
  → Suitable for: 100-1,000 users
  → Database size: ~5MB per 1,000 users
  → Query time: <10ms
  → Concurrent connections: 100+

To Scale to 10,000+ Users:
  1. Switch from SQLite → PostgreSQL
  2. Add caching layer (Redis)
  3. Use load balancer (Nginx/HAProxy)
  4. Implement database replication
  5. Set up monitoring (Prometheus/Grafana)

Performance Optimization:
  → Database connection pooling
  → API response caching
  → Lazy loading in frontend
  → CDN for static assets
  → Horizontal scaling with load balancer
